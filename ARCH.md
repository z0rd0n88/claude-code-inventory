## Overview

claude-code-inventory is a Claude Code plugin (v2.1.0, MIT) that auto-generates a unified `.CLAUDE.inventory.*` snapshot of every Claude Code automation — hooks, skills, plugins, agents, and MCP servers — organized by scope (global, project, local). The stack is pure Bash for hook scripts, Markdown with YAML frontmatter for skills and slash commands, and JSON for plugin/hook/eval configuration, with no external runtime dependencies. Control enters through two paths: a SessionStart hook (`hooks/inventory.cmd`/`inventory.sh`) that runs each session and emits a JSON staleness signal when the inventory is missing, older than 24 hours, or its config hash changed; and three slash commands (`/inventory`, `/inventory-global`, `/inventory-compare`) that trigger the core skill directly. The skill (`skills/inventory/SKILL.md`) is an ~800-line structured prompt Claude executes in seven phases (Discovery → Classification → Validation → Change Detection → Recommendations → Generation → Gitignore), producing `.CLAUDE.inventory.md`, `.CLAUDE.inventory.json`, and `.CLAUDE.inventory.hash`.
