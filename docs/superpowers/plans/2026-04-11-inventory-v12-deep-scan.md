# Inventory Plugin v1.2 "Deep Scan" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enrich the per-project inventory with memory files, scheduled tasks, plugin health, CLAUDE.md hierarchy chain, and hook execution order — plus JSON schema versioning.

**Architecture:** All 5 features are additive to the existing 7-phase SKILL.md structure. Each feature adds discovery sources (Phase 1), optional classification/validation rules (Phases 2-3), and output sections (Phase 6). The hook script (`inventory.sh`) gets updated hash inputs. No structural refactoring.

**Tech Stack:** Markdown (SKILL.md prompt), Bash (hook script), JSON (plugin metadata, evals)

**Key files:**
- `skills/inventory/SKILL.md` — core prompt (all 5 features modify this)
- `hooks/inventory.sh` — staleness detection hash (Task 7)
- `.claude-plugin/plugin.json` — version bump (Task 8)
- `evals/evals.json` — new test scenarios (Task 8)
